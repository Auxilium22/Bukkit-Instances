About

A Bukkit plugin which allows conversion of whole worlds to resettable dungeon instances. Also provided is a party system and commands for instance management. Originally created for dreamcraft.org. Features include the following.

  • Ability to convert a world into an instanced dungeon.
  • Multiple instances per world concurrently.
  • Instances reset a configurable time after the party leaves.
  • Abilty to allow non operators to create and edit command blocks.
  • A fully featured party system.
  • Instance aware /sethome command.
  • Full support for modifying monster spawners via the /ispawner command.
  • Command block editing support without requiring creating and operator.
  • PermissionsEx integration which allows world specific permissions to be automatically inherited by instances.
  • Multiverse-Inventories integration to allow configurable inventory sharing per instances.
  • Multiverse-Core integration to allow instance worlds to inherit Multiverse-Core settings from their parent world.
  • WorldEdit integration for portal area selection.
  • Vault integration for economy support.

Installation

Copy the jar to your plugins folder on your server. There are no hard depenencies, so everything should just work. The plugin has the following soft dependencies.

Multiverse-Inventories

If Multiverse-Inventories is present the plugin will integrate with the configuration and ensure than instances share inventories with their parent world.

Multiverse-Core

If Multiverse-Core is present, instance worlds will inherit all the Multiverse-Core settings of their parent world.

PermissionsEx

If present then instance worlds will inherit permissions from their parent world.

Vault

If present then you can set portal prices.

WorldEdit

If present then Instances uses the WorldEdit selection tool rather than it's own.

Commands

Commands are split into 3 sections, party commands (those commands pertaining to party usage), general commands (those commands pertaining to general actions) and portal commands (which relate to instance portal administration).

Party

Players can create, join, leave and administer their own parties with the following commands. Instanced worlds created for a party are only accessible to players in that party.

Command Permission Description
/pcreate <name> instances.party Create a party named <name>.
/pdisband instances.party Disband your party, normally you must be the leader of the party to do this, see Configuration.
/pleader <player> instances.party Assign leadership of the party to <player>.
/pkick <player> instances.party Assign leadership of the party to <player>, normally you must be the leader of the party to do this, see Configuration.
/pinvite <player> instances.party Invite <player> to join your party, normally any party member can do this, see Configuration.
/puninvite <player> instances.party Remove an invite for <player> to join your party, normally any party member can do this, see Configuration.
/pjoin <name> instances.party Join party <name> you have been invited to.
/pleave instances.party Leave the party you are a member of.
/p <text> instances.party Chat with other members of the party.
/pme <text> instances.party Emote privately to other members of the party.
/pinfo [<name>] instances.party Show information about your current party or party <name> if specified. Additional information is shown if you have the permission instances.party.info, see Permissions
/plist instances.party.list List current parties

General

The plugin includes several general commands which are often supplied by other plugins, these include motd and basic teleport commands, also a fully featured command to modify monster spawners is included. If you do not need the /home, /sethome, /spawn, /setspawn and /motd commands, download the bare version of the plugin.

Command Permission Description
/home instances.general.home Teleport to your home location.
/sethome instances.general.home Set your home to your current location. Note this command will not allow you to set your home inside an instance.
/spawn instances.general.spawn Teleport to the spawn world's spawn location.
/setspawn [none] instances.general.setspawn Set the spawn world to your current world. Note this command only sets the world, not the spawn location, use another plugin such as Multiverse for this. If none is specified it sets the spawn world to null.
/motd instances.general.motd Show the message of the day. File is found in plugins/Instances/motd.txt Use § to obtain colour codes.
/isave instances.general.save Save all Instances settings, such as homes, instance portals and home worlds. Typically saving is only performed when the plugin is unloaded at server shutdown.
/ireload instances.general.reload Load settings from disk overwriting those in memory. Any settings in memory will be overwritten.
/imob <type> [<count>] instances.general.spawnmob Spawn <type> mobs at the location you are looking at. 1 or <count> mobs are spawned. See mobs.txt for a list of valid mob names.
/igenocide <type> [remove] instances.general.genocide Kill all <type> mobs in the current world, if remove is specified, remove them from the world instead of killing them.
/icmd instances.general.cmd Turn the item in you hand into a command block.

/ispawner (create|[<type>]) [<flags>]

Permission: instances.general.spawner

Modify the spawner you are looking at, set the type spawned to <type>. See mobs.txt for a list of valid mob names. Flags are listed in the following table. If you are looking at a monster, turn it into a mob spawner at it's current location.

Flag Description
-i Show information about the spawner.
-m <delay> Set the minimum <delay> between spawns in server ticks (1/20th of a second). (default: 200)
-M <delay> Set the maximum <delay> between spawns in server ticks (1/20th of a second). (default: 800)
-s <count> Set the <count> of mobs to spawn. (default: 4)
-r <range> Set the <range> at which mobs can spawn from the spawner. (default: 4)
-p <range> Set the <range> which players must be within in order to activate the spawner. (default: 16)
-e <max> Set the <max> number of entities close by required to disable this spawner. (default: 6)
-I (hand|feet|legs|body|head) Set the item carried by spawned monsters to the item you are currently holding.
-d (hand|feet|legs|body|head):(always|<chance>) Set the drop chance of the carried item. Always means always drop the item, otherwise <chance> is a probability between 0 and 1, the item will be randomly damaged. e.g. -d head:0.75
-f <duration> Copy any potion effects you have to spawned monsters, granting them for <duration> in server ticks (1/20th of a second).
-h <health> Set the <health> of spawned monsters
-l (true|false) If true, spawned monsters will loot items, otherwise they will not.
-T (true|false) If true mobs created by this spawner will never despawn, otherwise they will. (default: false)
-v (true|false) If true mobs created by this spawner will be invulnerable, otherwise they will not. (default: false)
-L (true|false) For creepers, spawn powered if true, unpowered otherwise.
-t (normal|wither) For skeletons, set the skeleton type to normal or wither.
-z (<size>:random) For slimes and lava slimes, set the <size> of mobs spawned by this spawner. 1 = small, 2 = medium 4 = large, other values work but may result in unexpected behaviour. If random is specified slimes and lava slimes will have a random size. (default: random)
-P (<x,y,z>|none) Set the initial absolute world position of spawned monsters or if none remove it.
-V (<x,y,z>|none) Set the initial velocity of monsters spawned by this spawner. If none is specified unset it.
-a <type> Add <type> to the list of monsters that the spawner can spawn with a weighting of 100. See mobs.txt for a list of valid mob names. (Broken, spawners don't correctly save multiple mob spawns)
-S <index> Select a monster which the spawner spawns by <index>. (Broken, spawners don't correctly save multiple mob spawns)
-w <weight> Set the <weight> of the currently selected monster (default: 100). (Broken, spawners don't correctly save multiple mob spawns)
-R Remove the currently selected monster from the spawner. (Broken, spawners don't correctly save multiple mob spawns)

Portal

Finally the plugin includes commands for creating portals to access world instances. Portal creation is accomplished by using the tool (default: feather) to select a cuboid in the world where the portal resides, and in the world which you wish to create an instance. First you select the entrance portal area, and execute /isetentrance then select the portal area in the world you wish to instance and execute /isetdestination. Once this is done you can create your instance via /icreateportal <name>. Note: In a production environment you should unload the source world, if the source world saves a chunk at the same time as the destination world creates a chunk this may result in chunk corruption in the instance.

Command Permission Description
/iselectiontool instances.portal.tool Set the selection tool used to mark portals to the item you are currently holding. This is disabled if WorldEdit is installed. (default: feather)
/isetentrance instances.portal.create Set the entrance location of a subsequently created portal to your current selection.
/isetdestination instances.portal.create Set the destination location of a subsequently created portal to your current selection.
/icreateportal <name> instances.portal.create Create a portal named <name> with the previously selected entrance and destination worlds.
/ideleteportal <name> instances.portal.delete Delete the portal named <name>.
/iportallist instances.portal.list List the instance portals on the server.

/imodifyportal [(show|set|reset) <property> [<value>]]

Permission: instances.portal.modify

Show, set or reset various portal properties, which are listed in the following table. Item properties are set to what you are holding in your hand when you type the command, other properties you must specify a <value>.

Property Type Default Description
createPrice Number 0.0 The price in your currency for the first party member to pay in order to create the dungeon.
createItem Item None The item required for the first party member to pay in order to create the dungeon.
entryPrice Number 0.0 The price in your currency for all party members to pay in order to enter the dungeon (Note: if there is a createPrice, the first party member only pays that).
entryItem Item None The item required for party members to pay in order fot enter the dungeon (Note: if there is a createItem, the first party member only pays that).
recreateTime Duration 0 The cooldown in server ticks (1/20th of a second) which a player must wait before being able to create a dungeon instance with this portal (Note: they can still join other's parties and visit the instance during this time).
unloadTime Duration 0 If an instance is empty of party members for unloadTime then it will be deleted.
difficulty Difficulty normal Sets the difficulty for the instance, valid values are peaceful, easy, normal and hard.
defaultParty String null Sets the default party for the instance, if a player tries to enter the portal and they are not in a party, they will be added to the default party which will be created if needs be.
entrance.facing Facing null Sets the direction player face when they are teleported to the dungeon entrance portal. By default portals preserve the direction the player is facing, specifying no value sets it to the direction you are currently facing, north, south, east and west set it to compass directions.
destination.facing Facing null Sets the direction player face when they are teleported to the dungeon destination portal. By default portals preserve the direction the player is facing, specifying no value sets it to the direction you are currently facing, north, south, east and west set it to compass directions.

Configuration

There are only a handful of configuration options which should ever need to be modified manually, they are as follows.

PropertyDefaultDescription
partyNamePrefix §b[ Party name prefix, used in party messages.
partyNameSuffix ]§r Party name suffix, used in party messages.
leaderInvite false Allow only the party leader to invite members.
leaderKick true Allow only the party leader to kick members.
leaderDisband true Allow only the party leader to disband the party.
editCommandInCreative true Only allow editing command blocks in creative.

Example configuration

partyNamePrefix: '§b['
partyNameSuffix: ']§r'
leaderInvite: false
leaderKick: true
leaderDisband: true

Permissions

There are only a handful of permissions other than those required for commands, they are as follows.

Permission Description
instances.general.motd Show the message of the day on login.
instances.portal.create Allow usage of the cuboid selection tool, used with portal creation.
instances.party.info Show information about party instances in /pinfo.
instances.general.cmd.set.<command> Allow setting of command blocks to a specific command..
instances.general.cmd.set.* Allow setting of command blocks to any command..
instances.general.cmd.reset Allow setting of command blocks to an empty command.
